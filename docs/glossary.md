# Glossary

Project-specific terms a new user might not recognize. For deeper context, follow the "see" link on each entry.

## Living project model (`model.md`)

A per-repo markdown file at `~/.claude/presence/projects/<repo_id>/model.md` that Claude appends to across sessions. It captures Claude's understanding of the codebase: its layout, key abstractions, conventions, recent decisions. Re-injected at SessionStart so the next session starts with context. Compressed periodically by the `model-curator` subagent.

See: [architecture.md](architecture.md), [../skills/project-model/SKILL.md](../skills/project-model/SKILL.md).

## Outcome telemetry

Records what Claude committed (commit SHA, message) and watches for reverts, amends, and PR closes in subsequent sessions. Surfaces "X of your tracked commits were reverted since your last visit" in the SessionStart context. Stored in `~/.claude/presence/telemetry/`.

See: [architecture.md](architecture.md), `lib/telemetry.py`.

## Event digest

File-edit events, test pass/fail events, and bash command exits between turns are captured to `~/.claude/presence/events/<repo_id>/pending.jsonl` and surfaced at the next user prompt instead of needing to be polled. Drained on read.

See: `lib/events.py`.

## Calibrated confidence

The Stop hook checks the final assistant message for unhedged success claims ("fixed", "done", "works"). If found AND there has been an edit since the last passing test or build, it logs (or blocks, under stricter presets) to nudge against asserting completion without verification.

See: [architecture.md](architecture.md), `lib/verify.py`.

## Preset

A JSON file under `presets/` (or under `~/.claude/presence/presets/` for custom ones) selecting feature flags and thresholds. Four ship: `solo-dev`, `team-oss`, `enterprise-strict`, `zerotrust`. See the README presets table for the matrix; see [recipes.md](recipes.md) for custom-preset patterns.

See: [../README.md](../README.md) presets section.

## Zero-Trust preset

The opt-in profile that treats the plugin, the local user account, and the workspace as potentially compromised. Adds AES-GCM at-rest encryption, a tamper-evident audit log with per-line SHA-256 hash chain, SessionStart fail-closed integrity check, hard commit gates, settings immutability gated by `/presence-unlock`. Requires the `cryptography` library.

See: [zerotrust.md](zerotrust.md).

## Integrity manifest (`MANIFEST.lock`)

A SHA-256 hash of every shipped plugin file (`lib/`, `hooks/`, `presets/`, `commands/`, `agents/`, `skills/`, `.claude-plugin/`). Generated by `lib/integrity.py --write`. Verified on demand by `/presence-doctor`; under any preset with `integrity.fail_closed: true` (default in `zerotrust`), verified at SessionStart and the session goes inert on mismatch.

See: `lib/integrity.py`.

## Audit chain

The append-only log at `~/.claude/presence/audit.jsonl` written under the `zerotrust` preset. Each line includes the SHA-256 of the previous line so any tampering breaks the chain at the modified row and every row after. Verify with `python3 lib/integrity.py --audit-verify`.

See: [zerotrust.md](zerotrust.md), `lib/audit.py`.

## Pinned Python (`.python_bin`)

A marker file at `~/.claude/presence/.python_bin` containing the absolute path of a specific Python interpreter that hooks should use. Written by `install.sh --bootstrap` when uv installs a Python. Honored by `hooks/scripts/_common.sh` so hooks use the bootstrapped interpreter even when the user's PATH still points at an older `python3`. Falls through to PATH lookup when absent or stale.

See: [../install.sh](../install.sh), `hooks/scripts/_common.sh`.

## Hook event names

Claude Code lifecycle events that presence hooks bind to:

- **`SessionStart`** - new Claude Code session opening; presence injects living model + telemetry digest + recent events as `additionalContext`.
- **`UserPromptSubmit`** - user submitted a message; presence drains the events buffer into the next turn.
- **`PreToolUse`** (Bash) - about to run a shell command; presence inspects for `git commit`/`push` and gates on calibrated confidence.
- **`PostToolUse`** (Bash) - shell command finished; presence classifies the command (test pass/fail, build pass/fail) and logs an event.
- **`PostToolUse`** (Edit / Write / MultiEdit) - file change; presence logs an `edit` event.
- **`Stop`** - Claude is about to stop; presence parses the final message for unhedged success claims.

See: `hooks/hooks.json`, `hooks/scripts/`.

## Snapshot

A tar.gz of the state directory written by `install.sh --snapshot`. Excludes per-machine markers (`.python_bin`, cache markers, transient session state) and refuses under `zerotrust` (key portability is its own design problem; tracked in [roadmap.md](roadmap.md)). Restore with `install.sh --restore`. Schema-versioned via `_snapshot_meta.json`.

See: `lib/snapshot.py`, [recipes.md](recipes.md).

## Bug report bundle

The output of `/presence-bugreport` (or `python3 lib/bugreport.py --md`): a paste-friendly markdown blob combining install-verify output, doctor report, recent warnings, and state sizes. Designed to drop into the GitHub bug-report issue template.

See: `lib/bugreport.py`, `commands/presence-bugreport.md`.
